# Synthesize ArtStyle

You synthesize `ArtStyle` artifacts from Katagami curation jobs. The output must be driven by the user's query and curation direction. Do not use a fixed built-in ArtStyle taxonomy. Do not use web sourcing. Do not use procedural or rasterized fallback visuals. Do not publish geometric placeholders.

## Non-negotiable visual asset contract

Every visual asset attached to an `ArtStyle` must be generated by the platform's OpenAI Codex-authorized image generation capability.

Allowed visual provenance:

```text
openai_codex_image_generation
```

Disallowed visual provenance:

```text
online_source_image
web_search_image
curated_public_reference
procedural_preview
fallback_rasterized_composition
svg_placeholder
geometric_placeholder
manual_inline_asset
```

If OpenAI Codex-authorized image generation is unavailable, fail the curation job with a clear error. Do not publish a partial `ArtStyle`. Do not substitute online images, SVGs, procedural previews, or fallback compositions.

## Inputs

Read the `CurationJob` input and recover:

- `query_id`
- `direction_id`
- `task`
- `target_direction`
- `scope`
- `output_type`

Read the parent `CurationQuery` and `CurationDirection` for lineage. Do not perform web search for visual references. The style must be inferred from the user's intent and generated through image generation.

## Image generation requirements

Generate, at minimum:

- 3 reference images
- 4 proof shots covering distinct slots/use cases
- 1 thumbnail
- 1 published asset manifest referencing all generated images

For each generated image, store metadata:

- `asset_provenance = openai_codex_image_generation`
- `provider = openai_codex`
- `model` or subscription-backed generation engine name when exposed
- exact generation prompt
- negative prompt
- seed/request id/job id if available
- created timestamp
- parent `CurationQuery` id
- parent `CurationDirection` id
- parent `CurationJob` id

## Prompt policy

The `prompt_template` must be open-ended and query-derived. It must not default to:

- flat geometric article metaphor
- simple geometric shape system
- modular vector shapes
- oversized circles/rectangles/arrows
- generic Memphis shapes

The `negative_prompt` must include:

```text
default flat vector geometry, generic Memphis shapes, modular vector placeholders, empty geometric article metaphor, simple circles and rectangles, arbitrary arrows, procedural placeholder preview, SVG placeholder, UI wireframe, abstract layout blocks
```

## Quality gate

Before publishing, inspect the generated images. If the visual result reads as mostly geometric placeholder art, do not publish. Regenerate with a stronger pictorial prompt. If regeneration still fails, fail the job instead of publishing.

Publish only when:

- all visual assets are image-generation outputs
- provenance metadata is present
- no online/source/procedural/fallback assets are attached
- the images visibly express the queried art direction
- geometric placeholder bias is absent

## Entity action sequence

Use `ArtStyle` actions as available:

- `SetName`
- `SetMedium`
- `SetPromptTemplate`
- `SetSlotRecipes`
- `SetGuidance`
- `AttachReferenceImages`
- `VerifyReferenceImages`
- `AttachProofShots`
- `VerifyProofShots`
- `AttachThumbnail` or `AttachVerifiedThumbnail`
- `VerifyThumbnail`
- `AttachPublishedAssets`
- `SetLineage`
- `SubmitForReview`
- `UpdateQuality` / `MarkQualityPassed`
- `Publish`

Do not call `SetSources` with online source image references. If lineage fields require sources, use only internal generation provenance records, not web sources.

Then call curation completion actions with the produced `art_style_ids`. If image generation is unavailable, complete/fail the job with an explicit error rather than creating an invalid `ArtStyle`.
