# Katagami Design Principles

This file is the compact philosophy layer for Katagami. The reusable design
tests that used to live here have been extracted into Accepted `TasteRule`
records so synthesis and review agents can apply them consistently.

Use this document to understand the spirit of the system. Use Accepted
`TasteRules` for concrete pass/fail design guidance.

## Complete Visual Identity

A design language is a complete visual identity, not a decorative skin. It
should feel authored by a specific designer with a point of view. The strongest
languages have a recognizable structure, material logic, rhythm, and scene
grammar that survive beyond palette or font swaps.

## Restraint As Identity

Katagami values intentional absence. AI-generated interfaces often add colors,
shadows, gradients, rounded surfaces, and decorative elements to fill space.
Authored design earns identity through a narrow set of choices applied with
discipline.

Restraint does not mean sameness. It means every retained visual move must
carry structure, hierarchy, or product meaning.

## Structural Taste

Every choice should trace back to the language's philosophy: typography,
palette, spacing, border treatment, surface material, motion, and component
behavior. Defaults are acceptable only when they are deliberately chosen and
defended by the language.

The library should stay diverse. Strong rules protect quality without forcing
every language toward the same white SaaS surface, font family, or dashboard
scene.

## Scene-First Design

Embodiments should be plausible application screens for specific fictional
products. The product world should create real labels, constraints, artifacts,
states, and hierarchy. A generic component catalog is not enough to prove a
design language.

## Projection Discipline

Katagami specs are richer than `DESIGN.md`, but every language must still
project cleanly into portable design artifacts. The source spec, embodiment,
thumbnail, shadcn component recipes, and shadcn preview shots should all tell
the same visual story.
